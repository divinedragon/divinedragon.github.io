Install Apache Maven on Linux

August 1, 2020 - 5 mins read
  • maven
  • devops
  • tools

This article gives you a step by step walk-through for installing Apache Maven on Linux.

You can watch the entire installation demo in this video.

Introduction

Apache Maven is the most popular build and project management tool for Java projects. Maven is built on top of Java, so you need to have Java installed on your system.

Installation of Maven on any system consists of following 3 steps.

  • Download the Apache Maven binary zip from the official website.
  • Extracting the binary zip into a specific folder.
  • Configure the environment variables of the System.

Let me walk you through these steps in detail for a Linux system.

 

1. Downloading the Apache Maven

The first step is to download the Apache Maven from the official website. The downloads page for Apache Maven is this - https://maven.apache.org/download.cgi. On this page, you have two binary options to download - either a zip file or a tar.gz file (marked in blue boxes). Here is the screenshot below.

You can chose either of these. If you are not sure, chose the zip file. Also, download the associated checksum file (marked in orange boxes). You will need the checksum file to verify the zip or tar.gz file download.

You can use your browser to download the binary files, or you can use the wget or curl commands on the terminal to download these files.

Below example uses wget to download the zip file and the corresponding checksum file.

    
-> wget -q --show-progress https://downloads.apache.org/maven/maven-3/3.8.5/binaries/apache-maven-3.8.5-bin.zip
apache-maven-3.8.5-bin.zip            100%[============================>]   9.16M  13.5MB/s    in 0.7s

-> wget -q --show-progress https://downloads.apache.org/maven/maven-3/3.8.5/binaries/apache-maven-3.8.5-bin.zip.sha512
apache-maven-3.8.5-bin.zip.sha512     100%[============================>]     128  --.-KB/s    in 0s

-> ls -l
total 9384
-rw-rw-r-- 1 thedevjournal thedevjournal 9602303 Nov 19  2019 apache-maven-3.8.5-bin.zip
-rw-rw-r-- 1 thedevjournal thedevjournal     128 Nov 19  2019 apache-maven-3.8.5-bin.zip.sha512
 

2. Extracting the Binary archive File

2.1. Verifying the Checksum

Before you extract the zip or tar.gz file, perform the checksum validation. This ensures that there were no issues with the downloaded files.

The extension of the checksum file mentions the hash function of checksum. For apache-maven-3.8.5-bin.zip.sha512 checksum file, the extension is sha512. This indicates that checksum function used for apache-maven-3.8.5-bin.zip is sha512. Run the below command to generate the checksum for the binary file.

    
-> sha512sum apache-maven-3.8.5-bin.zip
1c095ed556... (removed for brevity) ...4eded94d  apache-maven-3.8.5-bin.zip

To perform checksum validation, you need sha512sum command to calculate the checksum of the zip file.

If you do not have sha512sum installed, you need coreutils package installed. Here is list of installation commands to install coreutils in various Linux distributions.

Verify the output of sha512sum and the contents of checksum file (apache-maven-3.8.5-bin.zip.sha512). If the values are same, the download was error-free.

    
-> sha512sum apache-maven-3.8.5-bin.zip
1c095ed556... (removed for brevity) ...4eded94d  apache-maven-3.8.5-bin.zip

-> cat apache-maven-3.8.5-bin.zip.sha512
1c095ed556... (removed for brevity) ...4eded94d

2.2. Installation Directory

You need to extract the zip file. Create an installation directory if it does not exist.

I prefer to keep all my installations in one directory (~/installed). You can chose to extract it elsewhere as well.

    
-> mkdir ~/installed
-> mkdir ~/installed/maven3

Move the downloaded binary file in the created maven3 directory. And now, cd into this directory so that you can extract the archive file here.

    
-> mv apache-maven-3.8.5-bin.zip ~/installed/maven3/
-> cd ~/installed/maven3
 

2.3. Extracting Binary Archive File

2.3.1. You downloaded the zip file (apache-maven-3.8.5-bin.zip)

You will need unzip command to extract the files inside the zip binary archive file. If you do not have unzip command installed, install it first. Here is list of installation commands to install unzip in various Linux distributions.

Run the following command to extract the zip file.

    
-> unzip apache-maven-3.8.5-bin.zip

After extracting the zip file, you do not need it, so delete the zip file.

    
-> rm apache-maven-3.8.5-bin.zip -f

2.3.2. You downloaded the tar.gz file (apache-maven-3.8.5-bin.tar.gz)

You will need tar command to extract the files inside the tar.gz binary archive file. If you do not have tar command installed, install it first. Here is list of installation commands to install tar in various Linux distributions.

Run the following command to extract the tar.gz file.

    
-> tar -xf apache-maven-3.8.5-bin.tar.gz

After extracting the tar.gz file, you do not need it, so delete the tar.gz file.

    
-> rm apache-maven-3.8.5-bin.tar.gz -f

2.4. Creating a Symbolic Link (Optional)

This is optional. I create symbolic links that always point to the latest binary version. For example, below is the output for all the Maven binary versions that I have downloaded and configured over time. The symbolic link latest is pointing to apache-maven-3.8.4.

    
drwxrwxr-x 2 thedevjournal thedevjournal 4096 Apr 26 01:24 apache-maven-3.6.3
drwxrwxr-x 6 thedevjournal thedevjournal 4096 Mar 11 23:49 apache-maven-3.8.4
lrwxrwxrwx 1 thedevjournal thedevjournal    5 Mar 11 23:53 latest -> apache-maven-3.8.4

This way, when I download a new version, I do not need to update the environment variables again. Environment variables always point to the latest Maven installations via symbolic link.

Here is how you will create a symbolic link to the extracted apache-maven-3.8.5 directory.

    
-> ln -s apache-maven-3.8.5 latest
 

3. Configure the Environment Variables

When you extracted the binary archive(zip or tar.gz), you have Maven installed. But, you needed to give the entire path (~/installed/latest/bin/mvn) to run your builds. To run mvn command, without specifying the entire path, you need to configure environment variables.

3.1. Configure M2_HOME

Most Java tools check for M2_HOME environment variable to locate Maven installation directory. You can configure this environment variable in .bash_profile, .bashrc, or .profile file.

Below is the entry that you need to add.

    
export M2_HOME="~/installed/maven3/latest"

3.2. Configure PATH Variable

To allow resolution of mvn command from any directory, configure PATH environment variable like below.

    
export PATH="$PATH:$M2_HOME/bin"

You need to re-import the environment variable for changes to take effect. Close the terminal window and open a new one.

4. Verify Maven Installation

Open a new terminal window and run mvn -version. You will see the command output the maven version and related information. This output indicates Maven is ready to start building your applications.

    
-> mvn -version

Apache Maven 3.8.5 (cecedd343002696d0abb50b32b541b8a6ba2883f)
Maven home: /home/thedevjournal/installed/maven3/latest
Java version: 11.0.10, vendor: Ubuntu, runtime: /usr/lib/jvm/java-11-openjdk-amd64
Default locale: en_US, platform encoding: UTF-8
OS name: "linux", version: "5.4.0-70-generic", arch: "amd64", family: "unix"
